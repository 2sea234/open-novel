# OpenNovel 测试说明

## 1. 测试范围

本项目目前主要针对以下模块进行接口与业务流程测试：

- gateway：JWT 鉴权、白名单、请求头透传
- admin-system：管理员、角色、菜单、权限、日志、个人中心
- novel-admin：分类、小说、TXT 导入、章节正文、回收站
- minio-image：图片上传、删除、文件类型校验
- novel-user：注册、登录、邮箱验证码等基础能力

## 2. 测试方式

当前测试主要通过以下方式完成：

- Postman 接口测试
- MySQL 数据库结果验证
- 日志输出检查
- 异常场景手动验证
- 前后端联调验证

## 3. 核心测试场景

| 模块 | 测试场景 | 预期结果 | 状态 |
|---|---|---|---|
| gateway | 不带 token 访问后台接口 | 返回 401 | 通过 |
| gateway | 携带错误 token 访问后台接口 | 返回 401 | 通过 |
| admin-system | 无权限用户访问受保护接口 | 返回 403 | 通过 |
| admin-system | 有权限用户访问受保护接口 | 正常返回 200 | 通过 |
| admin-system | 无效 roleId 分配角色 | 返回错误，原绑定不清空 | 通过 |
| admin-system | 删除 super_admin | 返回错误，不允许删除 | 通过 |
| admin-system | 删除有子菜单的菜单 | 返回错误，并记录失败日志 | 通过 |
| novel-admin | TXT 导入小说 | 正常生成文章、章节、段落、锚点 | 通过 |
| novel-admin | 删除文章后查询章节正文 | 查询不到已删除文章内容 | 通过 |
| minio-image | 上传正常图片 | 返回 URL 和 objectName | 通过 |
| minio-image | 上传伪造图片 | 拦截上传 | 通过 |

## 4. 典型测试用例

### 4.1 Gateway JWT 鉴权测试

| 项目 | 内容 |
|---|---|
| 测试目标 | 验证后台接口必须经过 JWT 鉴权 |
| 前置条件 | 已启动 gateway、admin-system |
| 测试数据 | 不携带 Authorization 请求后台接口 |
| 操作步骤 | 使用 Postman 请求 `/api/adminSystem/user/page` |
| 预期结果 | 返回 401 |
| 实际结果 | 返回 401 |
| 测试结论 | 通过 |

### 4.2 RBAC 无权限访问测试

| 项目 | 内容 |
|---|---|
| 测试目标 | 验证无权限用户不能绕过前端直接调用接口 |
| 前置条件 | 普通管理员已登录，但未分配删除用户权限 |
| 测试数据 | 普通管理员 token |
| 操作步骤 | 使用 Postman 调用删除用户接口 |
| 预期结果 | 返回 403 |
| 实际结果 | 返回 403 |
| 测试结论 | 通过 |

### 4.3 无效 ID 防误清空测试

| 项目 | 内容 |
|---|---|
| 测试目标 | 验证分配权限时传入无效 ID 不会清空原绑定 |
| 前置条件 | 角色已有权限绑定 |
| 测试数据 | `permissionIds: [999999]` |
| 操作步骤 | 调用角色分配权限接口 |
| 预期结果 | 返回错误，原权限绑定仍保留 |
| 实际结果 | 返回错误，重新查询原权限仍存在 |
| 测试结论 | 通过 |

### 4.4 TXT 导入测试

| 项目 | 内容 |
|---|---|
| 测试目标 | 验证 TXT 文件能被解析成文章、章节、段落和锚点数据 |
| 前置条件 | 准备合法 TXT 文件 |
| 测试数据 | 小说 TXT 文件 |
| 操作步骤 | 调用 TXT 导入接口 |
| 预期结果 | articles、chapters、paragraphs、paragraph_anchors 正常写入 |
| 实际结果 | 数据正常写入，章节和段落顺序正确 |
| 测试结论 | 通过 |
